# Superpowers Handoff

What Fable built in Phase 0, and how to turn it into the full
`trading-office` application.

## What exists

- **`packages/office-visual-kit`** — reusable floor renderer:
  Tiled loader/normalizer, scene schema + validation, asset registry,
  tile/agent/object/badge renderers, interaction manager, pixi-viewport
  camera, React wrapper (`OfficeSceneCanvas`, `useOfficeScene`). Core is
  plain PixiJS v8; React is wrapper-only (acceptance criterion).
- **`examples/trading-lab-research-floor`** — runnable preview
  (`npm install && npm run dev` from the repo root → http://localhost:5173):
  7 agents (Boss/Orchestrator, Strategy Analyst, Researcher, Critic, Builder,
  Evaluator, Performance Monitor), interactive objects (Boss Console, Wall
  Monitor, Hypothesis Board, Bot Status Monitor, Archive Shelf, Server Rack,
  Holo Data Table), hover/click debug card, preview-side status simulation.
- **Asset pipeline** — original CC0-equivalent pixel art generated by
  deterministic scripts (`tools/`), license policy documented.
- **Map pipeline** — valid Tiled `.tmj` (generated by a reviewable script,
  editable in Tiled), conventions documented.

## How to run / verify

```bash
npm install
npm run dev        # preview at :5173
npm run build      # typecheck + production build
npm run generate   # regenerate assets + map
```

## Integration sketch for the production app

```text
apps/web (React 19)
  └─ <OfficeSceneCanvas config={floorConfig}
        onAgentClick={openAgentPanel}
        onObjectClick={(o) => openPanel(o.panelTarget)}
        onSceneReady={(scene) => sceneRef.current = scene} />

apps/server (office gateway)
  └─ pushes agent/workflow state → web → scene.setAgentStatus(id, status)
```

- `ObjectEntity.panelTarget` is the routing key for panels — the example
  already assigns `backtest-summary`, `hypothesis-pipeline`, `bot-health`,
  `knowledge-base`, `infra-status`, `boss-commands`, `shared-context`.
- Multiple floors = multiple `OfficeSceneConfig`s (one `.tmj` + config per
  agent system). The elevator/floor-switcher is app chrome, not kit code.
- The kit is consumed as TypeScript source via workspace exports
  (`.` and `./react`). If you need a publishable build, add a `tsc`/bundler
  emit — nothing in the source blocks it.

## Intentionally missing (do not look for it)

- No backend, no Hono, no Postgres, no auth, no Docker Compose.
- No real `trading-lab` API client, no SSE/WebSocket — the status simulation
  in the preview is a `setInterval` stand-in for the future gateway stream.
- No production dashboard panels — debug card only.
- No execution authority anywhere in this repo (architecture rule:
  `trading-office` visualizes and routes commands; `trading-platform`
  executes).
- No drag-and-drop floor editor — Tiled is the editor.

## Recommended next steps

1. **apps/web shell**: floor switcher + panel router around
   `OfficeSceneCanvas`; replace the debug card with real panels keyed by
   `panelTarget`.
2. **Office gateway (apps/server)**: read-only adapters to `trading-lab`
   (mock mode first), stream agent statuses; wire to `setAgentStatus` and a
   future `scene.setObjectBadge`-style API if objects need live markers.
3. **Status vocabulary**: confirm the `AgentStatus` union against real
   trading-lab workflow states; extend `DEFAULT_STATUS_COLORS` accordingly.
4. **More floors**: author the next agent system as `.tmj` + scene config —
   that path is the kit's whole point and needs no renderer changes.
5. **Art pass**: replace generated placeholders role by role (see
   asset-guidelines.md); the registry isolates art from code.
6. **Perf, if floors grow**: tile layers are plain sprite containers; if a
   floor exceeds a few thousand tiles, add `cacheAsTexture` on static layers
   inside `renderTileLayers` — measure first.

## Known sharp edges

- `OfficeSceneCanvas` recreates the scene when `config` identity changes —
  memoize configs.
- The example's `main.tsx` skips `<StrictMode>` (double-mounted effects vs
  async Pixi init in dev); the wrapper cleans up correctly either way, but if
  you enable it, verify there's no canvas flicker in dev.
- Tile data must be CSV or uncompressed base64; embedded tilesets only —
  loader errors are explicit about both.
- `pixi.js`, `react` and the kit must be deduped to single versions in the
  final app bundle (the example's vite config shows `resolve.dedupe`).
