# trading-office

AI Office Shell / Control Room for multiple agent systems.

`trading-office` is a visual frontend / control-room layer. It has **no execution
authority**: `trading-platform` stays the execution/data authority, `trading-lab`
is the first connected agent system ("first floor").

## Phase 0 — Office Visual Builder Kit

This repository currently contains Phase 0 only: a reusable visual constructor
for pixel-art office floors plus one reviewable example floor.

```text
Tiled map → scene schema / semantic config → PixiJS renderer → React preview wrapper
```

| Path | What it is |
| --- | --- |
| `packages/office-visual-kit/` | The kit: PixiJS v8 renderer core, Tiled loader, scene schema, asset registry, interaction, camera, React wrapper |
| `examples/trading-lab-research-floor/` | Runnable preview: Trading Lab Research Floor (7 agents, interactive objects, debug overlay) |
| `packages/office-visual-kit/docs/` | Kit documentation incl. Tiled conventions and the Superpowers handoff |
| `docs/fable-input/` | The original input/spec pack for this phase |

## Quick start

```bash
npm install
npm run dev
```

Open http://localhost:5173 — you should see the pixel-art Trading Lab Research
Floor. Drag to pan, scroll to zoom, hover/click agents and objects to inspect
them in the debug card.

Other commands:

```bash
npm run build      # typecheck + build kit and example
npm run generate   # regenerate placeholder pixel assets + the Tiled map
```

## Visual style & licensing

The visual style ("Retro Pixel AI Research Tower") is original. All pixel
assets in the example are generated by scripts in this repository
(`examples/trading-lab-research-floor/tools/`) and are CC0-equivalent.
Prototype environment assets may later include Kenney CC0 assets. Reference
screenshots were used only as private mood references and were not copied into
the final layout, characters, or assets.

See `packages/office-visual-kit/docs/asset-guidelines.md` for the full policy.

## What is intentionally NOT here (Phase 0)

No backend (Hono), no Postgres, no auth, no real `trading-lab` API, no
SSE/WebSocket, no production dashboard panels, no Docker Compose. Those come
later — see `packages/office-visual-kit/docs/superpowers-handoff.md`.
